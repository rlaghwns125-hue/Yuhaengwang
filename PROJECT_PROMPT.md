# Yuhaengwang (유행왕)

## 프로젝트 개요

실시간 디저트 트렌드 순위를 보여주고, 클릭하면 관련 카페/시설을 지도에 표시하는 크로스플랫폼(웹 + 앱) 서비스.
로그인/회원가입 지원. 회원은 검색 이력 기반 AI 개인화 순위 제공.

---

## 메인 화면 레이아웃

- 상단: 검색창 + 로그인/회원가입 버튼
- 배경: 네이버 지도 (전체 화면)
- 하단 오버레이: 디저트 트렌드 버튼들 (아이콘 + 이름, 횡스크롤)
- 버튼 클릭 시 해당 디저트 관련 카페가 지도에 마커로 표시

---

## 핵심 기능

### 1. 비회원 모드

- AI가 학습한 전체 트렌드 기반으로 인기 높은 순서대로 디저트 버튼 표시
- 버튼 형태: (디저트 아이콘) + 디저트명
- 클릭하면 현재 위치 기준 관련 카페를 지도에 마커로 표시
- 검색창으로 직접 디저트/카페 검색 가능

### 2. 회원 모드

- 로그인/회원가입 (이메일 또는 소셜 로그인)
- 개인화 AI 순위:
  - 회원의 검색 이력을 AI가 학습
  - 자주 검색한 디저트가 우선순위 높게 표시
  - 단, 한두 번 검색으로 바로 최상위가 되지 않음
  - 가중치 점수 시스템: 검색 횟수 + 검색 빈도(최근성) + 반복 패턴을 종합하여 점수 산출
  - 일정 임계값 이상이 되어야 순위에 반영 (단골 패턴 감지)
- 개인화 순위 이후에는 비회원과 동일한 전체 트렌드 순위 표시

### 3. 개인화 점수 알고리즘

> 개인화 점수 = (검색 횟수 x 횟수 가중치) + (최근 검색 빈도 x 빈도 가중치) + (연속 검색일 보너스)

- "크로플" 3회 검색 (최근 1주) -> 점수 낮음 -> 아직 트렌드 순위 유지
- "크로플" 15회 검색 (최근 1달, 꾸준히) -> 점수 높음 -> 개인 순위 상위로 이동
- 오래 검색 안 하면 점수 자연 감소 (시간 감쇠)

### 4. 지도 기반 장소 표시

- 네이버 지도 SDK (백그라운드 전체 화면)
- 디저트 버튼 클릭 or 검색하면 현재 위치 기준 관련 카페/시설 마커 표시
- 마커 클릭하면 카페 정보 카드 (이름, 주소, 리뷰, 사진)
- 거리순 / 평점순 정렬

### 5. 실시간 트렌드 수집

- 네이버 데이터랩 API + SNS 크롤링으로 디저트/놀이 트렌드 수집
- AI(Claude API)로 디저트 관련 키워드만 필터링 + 순위 산출
- 주기적 업데이트 (1시간~하루 간격)
- DB에 캐싱하여 API 호출 최소화

## 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React Native + Expo (Web 지원) | 웹/앱 동시 배포 |
| 지도 | 네이버 지도 SDK | 웹: JS SDK / 앱: 네이티브 SDK |
| 백엔드 | Firebase Cloud Functions | 트렌드 수집 + API 중계 (서버리스) |
| 인증 | Firebase Auth | 이메일/구글/카카오 소셜 로그인 |
| DB | Cloud Firestore | 트렌드/검색이력/개인화점수 |
| 트렌드 데이터 | 네이버 데이터랩 API + SNS 크롤링 | 실시간 검색 트렌드 |
| AI 분석 | Claude API | 트렌드 분류 + 개인화 순위 산출 |
| 장소 검색 | 네이버 지역 검색 API | 키워드 기반 주변 장소 검색 |
| 언어 | TypeScript | 프론트 + 백엔드 공용 |

## 아키텍처

**프론트엔드 (React Native Web)** - 웹/앱 동시 지원
- 검색창
- 디저트 트렌드 버튼
- 네이버 지도 (배경)

-> API 호출

**Firebase**
- Firebase Auth: 로그인/회원가입
- Cloud Functions: 트렌드 수집/AI 분석, 장소 검색 API 중계, 개인화 점수 산출
- Cloud Firestore: 사용자, 검색 이력, 개인화 점수, 트렌드, 장소 캐시

---

## Firestore 컬렉션 구조

**users/{uid}**
- email, provider (email/google/kakao), createdAt

**users/{uid}/searchHistory/{docId}**
- keyword, searchedAt

**users/{uid}/preferences/{keyword}**
- score, searchCount, lastSearched, updatedAt

**trends/{docId}**
- keyword, icon, rank, score, updatedAt

**places/{docId}**
- name, address, latitude, longitude, category, rating, imageUrl, cachedAt

## 화면 구성

**메인 화면 (지도 + 트렌드)**
- 검색창 (상단)
- 로그인/회원가입 버튼 (상단 우측)
- 네이버 지도 (전체 배경)
- 디저트 트렌드 버튼 (하단 오버레이, 횡스크롤)
- 버튼 클릭시 지도에 카페 마커 표시, 마커 클릭시 카페 정보 카드

**로그인/회원가입**
- 이메일 로그인
- 소셜 로그인 (구글, 카카오)
- 회원가입 폼

**회원 전용**
- 개인화 순위가 트렌드 버튼 앞쪽에 우선 표시

## 필요한 외부 API 키

- 네이버 지도 SDK (Client ID)
- 네이버 지역 검색 API (Client ID / Secret)
- 네이버 데이터랩 API (Client ID / Secret)
- Claude API 키 (트렌드 분석 + 개인화)
- Firebase 프로젝트 (Auth + Firestore + Cloud Functions)

---

## 제약 및 주의사항

- 외부 API 의존이라 인터넷 연결 필수
- 네이버 API 무료 티어 일일 호출 제한 있음 -> 캐싱 전략 필요
- 트렌드 데이터는 주기적 배치로 수집하여 DB에 캐싱
- 개인화 점수는 시간 감쇠 적용 -> 오래 검색 안 하면 점수 자연 하락
- 지도 SDK가 웹/앱에서 다르므로 플랫폼별 분기 처리 필요
- 비회원도 핵심 기능(트렌드 + 지도) 사용 가능해야 함
